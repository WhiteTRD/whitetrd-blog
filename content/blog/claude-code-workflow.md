## CLAUDE.md — мозок проєкту

Файл `CLAUDE.md` в корені репозиторію — це інструкції для агента. Він читає їх на початку кожної сесії.

Що включати:

- Команди для збірки/тестів
- Архітектуру проєкту
- Ключові конвенції
- Що НЕ робити

{{< callout type="warning" >}}
Не пишіть в CLAUDE.md те, що можна дізнатися з коду. Фокус на неочевидних рішеннях і конвенціях.
{{< /callout >}}

## Hooks

Автоматичні дії при подіях:

- **PreToolUse** — валідація перед виконанням інструменту
- **PostToolUse** — реакція на результат
- **Stop** — фінальна перевірка перед завершенням

## GitHub Projects як пам'ять

Issues + Project board = persistent memory між сесіями. Детальніше в [окремому пості](/blog/github-projects-agent-memory/).

## Головне правило

Чим більше контексту ви даєте агенту, тим точніше результат. Інвестуйте час у CLAUDE.md і описи issues — це окупиться.
